# MCP Tool Use

Инструменты поступают из MCP-серверов, перечисленных в `mcp/mcp.config.json`,
и фильтруются списком `mcp/tools.manifest.yaml`. Ты обязан:

1. **Доверять только whitelisted-инструментам.** Если LLM-ответ пытается вызвать инструмент,
   которого нет в манифесте, — отмени вызов и сообщи об ошибке.
2. **Читать схему аргументов.** Перед вызовом сверь имена/типы полей со схемой, пришедшей
   от MCP-сервера. Заполняй только описанные поля.
3. **Минимизировать вызовы.** Если тот же результат можно получить из краткой памяти
   или предыдущего наблюдения — не вызывай инструмент повторно.
4. **Ограничивать цепочку.** Максимум `tool_call_budget` вызовов на один пользовательский
   запрос (значение в `agent.yaml`, по умолчанию 12). При превышении — останавливайся
   и запрашивай у пользователя/клиента уточнение.
5. **Обрабатывать ошибки.** На ошибку инструмента: (a) один безопасный ретрай с экспоненциальным
   backoff, (b) при повторной ошибке — вернуть диагностическое сообщение пользователю.
6. **Приватность.** Никогда не передавай секреты (`.env`, пароли, токены) в аргументах
   инструментов. Если инструмент требует токен — он приходит через переменные окружения
   MCP-сервера, а не через модель.

Формат вызова (function-calling): стандартный OpenAI tools API. MCP-клиент автоматически
мапит `tool_name` → `(server_id, tool_id)`.
